# ai-systems-marketplace

Магазин плагинов для **Claude Code**. Раздаёт **споры** — плагины, которые одной командой разворачивают готовую систему управления проектом: память, правила, автоматизацию и «ризоматический паспорт» (родословную системы).

Ниже — как поставить.

## Установка

Спора ставится через **Claude Code** — это «рабочий» режим Claude, в котором он умеет выполнять команды на вашем компьютере. Хорошая новость: открывать терминал и печатать команды вручную не нужно — достаточно попросить Claude словами.

### Способ 1 — через приложение Claude, режим «Код» (проще всего)

Если вы пользуетесь приложением Claude для Mac — действуйте так:

1. Откройте приложение и переключитесь в режим **«Код»** (Code). Если он попросит выбрать папку — выберите ту, где будет жить проект (например, «Документы»).
2. Напишите Claude обычными словами, например:
   > Поставь, пожалуйста, плагин `project-setup` из GitHub-маркетплейса `mrdmitrydouble/ai-systems-marketplace`.
3. Claude начнёт установку и будет показывать кнопку **«Разрешить»** (Approve) — нажимайте её. Печатать команды самому не нужно.
4. Когда Claude скажет, что плагин установлен, — перезапустите приложение (или попросите: «перезагрузи плагины»).
5. Запустите разворачивание системы — напишите словами:
   > Инициализируй проект. Я получил спору от Дмитрия.

Готово — Claude создаст структуру памяти, правила работы и автоматику нового проекта.

⚠️ Это работает именно в режиме **«Код»**, потому что там Claude может выполнять команды. В режиме «Чат» установка плагинов недоступна.

### Способ 2 — командами (если вы уже работаете в Claude Code)

Введите эти две строки по очереди и нажмите Enter после каждой:

```
/plugin marketplace add mrdmitrydouble/ai-systems-marketplace
/plugin install project-setup@ai-systems-marketplace
```

Затем введите `/reload-plugins` (или перезапустите Claude Code) и запустите инициализацию словами: «Инициализируй проект, я получил спору».

Если Claude отвечает, что команда `/plugin` ему не знакома, обновите Claude Code до последней версии.

## Плагины

| Плагин | Версия | Что делает |
|--------|--------|-----------|
| `project-setup` | 8.0.0 | Инициализация мета-проекта или подопечного; миграция между версиями. Скилл + 6-слойный harness + шаблоны. Латинские (ASCII) имена папок и файлов. |

## Обновление (доставка новой версии)

Когда выходит новая версия споры — обновить уже установленный плагин:

```
/plugin marketplace update ai-systems-marketplace
/plugin update project-setup@ai-systems-marketplace
```

### ⚠️ Переход на v8 (breaking change)

В версии 8.0.0 имена папок и файлов структуры стали **латинскими** (`Project/`, `System/memory/`, `Documents/Drafts/` вместо `Проект/`, `Система/память/`, `Документы/Черновики/`) — среда отвергала не-ASCII имена в спорах.

- **Новые проекты** разворачиваются сразу на латинской структуре — делать ничего не нужно.
- **Проекты, развёрнутые на старых (кириллических) версиях,** продолжают работать со своей структурой: обновление плагина не переименовывает уже созданные папки. Чтобы перейти на латиницу — попросите агента провести миграцию по `MIGRATION.md` (отдельная операция с бэкапом).

## Для мейнтейнера (Дмитрий)

**Источник истины** скилла — мета-проект: `Создание ИИ систем/Проект/Документы/Скиллы/project-setup/`. Этот marketplace — артефакт сборки (как раньше ZIP).

**Цикл выпуска новой версии:**
1. Отредактировать скилл в мета-проекте, прогнать ревью (адверсариальное) + smoke-тесты harness.
2. Синхронизировать в плагин: `cp -R "…/project-setup/." plugins/project-setup/skills/project-setup/`
3. Поднять `version` в `plugins/project-setup/.claude-plugin/plugin.json` И в `.claude-plugin/marketplace.json` (держать синхронно — при расхождении plugin.json выигрывает молча).
4. **Обновить таблицу версий в этом README** (секция «Плагины»). При бампах 7.0.1→7.0.3 её забывали — она отставала; теперь это явный пункт чек-листа.
5. **Линт ASCII-имён:** `bash scripts/check-spore-ascii.sh` → должен быть `exit 0`. Не-ASCII имя файла/папки в `plugins/` → загрузчик отвергает плагин («invalid characters»). С v8 даже шаблоны структуры — латиницей. Содержимое файлов кириллицей — можно; имена — только ASCII.
6. `git commit + push`. Узлы получают обновление через `/plugin update`.

**Доступ:** репозиторий публичный — любой может подключить marketplace и поставить спору. Приватных данных в спорах нет (это шаблоны системы).
